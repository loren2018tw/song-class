## MODIFIED Requirements

### Requirement: 教師端必須提供小白板切換入口與主畫面配置

教師端頁面 MUST 在學生清單上方提供「首頁」與「小白版」功能按鈕，教師可在同一頁切換目前功能主畫面，並同步控制學生端顯示模式。

#### Scenario: 教師切換至小白板模式

- **WHEN** 教師在教師端頁面點擊「小白版」按鈕
- **THEN** 系統 MUST 切換到小白板主畫面
- **AND** 系統 MUST 將學生清單保留在左側區域
- **AND** 系統 MUST 將小白板顯示在右側主要內容區
- **AND** 系統 MUST 對所有已連線學生廣播白板模式事件

#### Scenario: 教師切換回首頁模式

- **WHEN** 教師在教師端頁面點擊「首頁」按鈕
- **THEN** 系統 MUST 關閉教師端小白板主畫面並回到首頁主畫面
- **AND** 系統 MUST 對所有已連線學生廣播首頁模式事件
- **AND** 學生端 MUST 同步顯示原本「請專心學習」首頁畫面

#### Scenario: 切回首頁時保留白板快照狀態

- **WHEN** 教師由白板模式切換回首頁模式
- **THEN** 教師端 MUST 保留目前白板快照於記憶體狀態
- **AND** 學生端 MUST 保留最近一次已套用的白板快照於記憶體狀態
- **AND** 系統 MUST 不因模式切回首頁而清空白板既有繪製資料

#### Scenario: 再次切回白板時使用既有快照恢復

- **WHEN** 教師由首頁模式再次切換為白板模式
- **THEN** 教師端 MUST 直接以既有白板快照恢復原畫面
- **AND** 學生端 MUST 依既有快照恢復白板畫面
- **AND** 教師端 MUST 不需重新傳送整份白板資料即可完成恢復

#### Scenario: 教師在同頁瀏覽學生清單與功能列

- **WHEN** 教師在教師端頁面停留並切換功能模式
- **THEN** 系統 MUST 保持上方為功能按鈕列
- **AND** 系統 MUST 保持下方左側為學生清單
- **AND** 系統 MUST 保持下方右側為目前功能主畫面區

### Requirement: 小白板元件必須輸出可重現的 JSON 塗繪資料

小白板元件 MUST 將塗鴉內容整理為可序列化 JSON，並同時提供可用於遠端同步的增量事件資料；背景資料 MUST 與塗鴉筆畫資料分離表示。

#### Scenario: 塗繪內容更新時回傳增量與最新重現資料

- **WHEN** 使用者完成一段塗繪、切換工具或清除畫布
- **THEN** 小白板元件 MUST 產生對應增量事件供即時傳送
- **AND** 小白板元件 MUST 保留可供新加入者初始化的最新重現快照

#### Scenario: 背景資料更新時保留分離結構

- **WHEN** 父元件變更白板背景圖片資料並由元件套用成功
- **THEN** 小白板元件 MUST 在可重現資料中保留目前背景資訊
- **AND** 背景資訊 MUST 與塗鴉筆畫分開儲存

#### Scenario: 新加入者需要重現既有內容

- **WHEN** 父元件偵測有新加入學生完成連線
- **THEN** 系統 MUST 能從白板元件取得最新快照並傳送給該學生
- **AND** 該學生 MUST 可在接收後重現加入前的白板內容
