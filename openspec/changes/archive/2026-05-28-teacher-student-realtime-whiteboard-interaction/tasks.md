## 1. 白板模式與分頁協定擴充

- [x] 1.1 在白板同步型別新增分頁欄位（教師白板/學生白板）與版本欄位
- [x] 1.2 擴充 DataChannel 訊息路由，區分教師下行事件與學生上行批次事件
- [x] 1.3 為學生上行批次事件加入學生識別、序號檢查與缺口處理策略

## 2. 教師端雙分頁與廣播控制

- [x] 2.1 在 WebTeacherView 建立白板區 `v-tabs`（教師白板、學生白板）並保留首頁切換
- [x] 2.2 實作教師切換分頁時的本地狀態同步，且不強制同步學生端分頁
- [x] 2.3 確保教師切回首頁時保留教師白板與學生白板快照狀態
- [x] 2.4 確保教師高頻切換兩個白板分頁時不重建元件且不遺失資料

## 3. 學生端受控雙分頁與繪製限制

- [x] 3.1 在 StudentView 實作雙分頁渲染：教師白板唯讀、學生白板可繪圖
- [x] 3.2 開放學生端手動切頁，且不受教師切頁強制覆寫
- [x] 3.3 在非學生白板分頁阻擋本地繪製輸入與上傳

## 4. 學生上行批次同步與教師重播

- [x] 4.1 學生端在學生白板分頁輸出批次增量事件並上傳教師端
- [x] 4.2 教師端按學生連線識別維護獨立白板快照與最後套用序號
- [x] 4.3 教師端重播學生批次事件並在序號異常時觸發重同步或重置

## 5. 教師端學生白板總覽 UI

- [x] 5.1 建立教師端學生白板總覽網格，依在線人數自動縮放縮圖尺寸
- [x] 5.2 以白板元件唯讀模式渲染每位學生縮圖，隱藏工具列並顯示學生暱稱標題
- [x] 5.3 在學生加入/離線時即時新增或移除對應縮圖且不影響其他縮圖
- [x] 5.4 僅實作清空全部學生白板，不提供清空單一學生白板

## 6. 驗證與回歸

- [x] 6.1 驗證教師在教師白板與學生白板分頁切換時，不會強制改變學生端分頁
- [x] 6.2 驗證學生白板塗鴉會批次回傳並在教師端正確重播
- [x] 6.3 驗證教師端可同時顯示全班縮圖且多人連線下仍保持可接受流暢度
