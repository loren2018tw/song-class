## Why

教師端目前缺乏可同時支援「聯絡簿顯示」與「作業繳交控管」的統一任務管理流程，教師需要在不同介面或以人工方式追蹤，容易造成重工與遺漏。此變更將同一筆任務資料同時驅動聯絡簿與繳交管理，降低操作成本並提升課務追蹤效率。

## What Changes

- 新增教師端「聯絡簿管理」選單，頁面包含「聯絡簿」與「作業繳交管理」兩個標籤。
- 新增統一任務資料模型，欄位包含：id、班級id、日期、任務名稱、顯示在聯絡簿、是否需控管、已完成。
- 新增任務新增、編輯、刪除流程，並在兩個標籤頁皆提供「新增任務」按鈕（功能一致）。
- 建立顯示規則：勾選「顯示在聯絡簿」則出現在聯絡簿標籤；勾選「是否需控管」則出現在作業繳交管理標籤；兩者可同時勾選但不可同時未勾選。
- 新增需控管任務的學生繳交狀態追蹤，可區分已繳交與未繳交。
- 新增作業管理清單篩選切換：顯示全部、未完成、已完成。

## Capabilities

### New Capabilities

- `contact-book-task-management`: 教師可在聯絡簿管理頁建立、編輯、刪除任務，並用同一份任務資料驅動聯絡簿與作業管理顯示。
- `assignment-submission-tracking`: 針對需控管任務記錄每位學生的繳交狀態，並顯示已繳交與未繳交。
- `assignment-completion-filter`: 作業繳交管理可依任務完成狀態切換顯示全部、未完成、已完成。

### Modified Capabilities

- 無

## Impact

- 前端：教師端 Web UI（主要為 WebTeacherView 與相關 Vue 元件）需新增聯絡簿管理雙標籤、任務操作與篩選介面。
- 後端：Rust 端需新增或擴充任務資料存取 API 與繳交狀態查詢/更新邏輯。
- 資料層：新增任務與任務-學生繳交狀態資料表設計與查詢流程（開發期依 v0 DDL 與命令列方式調整結構）。
- 即時同步：教師端更新任務或繳交狀態後，需與既有課堂會話資料流整合以維持畫面一致。
