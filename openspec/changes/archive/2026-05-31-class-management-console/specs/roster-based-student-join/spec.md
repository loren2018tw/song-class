## ADDED Requirements

### Requirement: 學生端必須顯示目前班級學生名單按鈕供加入

系統 MUST 在學生端加入畫面顯示目前班級學生名單，並以每位學生一個按鈕供選擇加入。

#### Scenario: 學生端載入後顯示名單按鈕

- **WHEN** 學生使用瀏覽器進入學生端頁面
- **THEN** 系統 MUST 顯示目前班級全部學生按鈕
- **AND** 每個按鈕 MUST 以「座號暱稱（無分隔）」格式顯示學生名稱（例如 `01王小明`；若暱稱為空則僅顯示座號）

### Requirement: 網頁介面顯示學生名稱時必須統一為座號暱稱

系統 MUST 在所有 Web 介面中，將原本顯示學生暱稱的位置統一顯示為「座號暱稱」。

#### Scenario: 教師端與學生端列表顯示一致格式

- **WHEN** 任一 Web 頁面需要顯示學生名稱
- **THEN** 系統 MUST 顯示「座號暱稱（無分隔）」
- **AND** 不得僅顯示暱稱而省略座號

#### Scenario: 暱稱為空時顯示規則

- **WHEN** 學生資料暱稱為空字串
- **THEN** 系統 MUST 至少顯示座號
- **AND** 系統 MUST 不顯示 `null`、`undefined` 或類似佔位文字

### Requirement: 已連入學生按鈕必須鎖定不可再次加入

系統 MUST 依即時連線狀態將已在線學生按鈕設為不可操作，避免同一名單項目重複加入。

#### Scenario: 名單項目已在線時按鈕 disabled

- **WHEN** 某位學生已成功連入教室
- **THEN** 該學生對應按鈕 MUST 顯示為不可點擊
- **AND** 其他學生按鈕 MUST 仍維持可操作狀態

#### Scenario: 已鎖定按鈕被點擊

- **WHEN** 使用者嘗試點擊已 disabled 的學生按鈕
- **THEN** 系統 MUST 不送出新的加入請求
- **AND** 頁面 MUST 維持目前加入流程狀態

### Requirement: 學生選擇名單項目後必須建立對應身分

系統 MUST 在學生點選可用名單按鈕時建立與該班級學生項目一致的會話身分。

#### Scenario: 選擇可用學生按鈕成功加入

- **WHEN** 學生點選一個可用的名單按鈕
- **THEN** 系統 MUST 建立該班級與座號對應的會話身分
- **AND** 系統 MUST 回傳加入成功並更新該按鈕狀態為不可操作

#### Scenario: 班級切換後學生需重新加入

- **WHEN** 教師切換目前班級而學生被強制登出
- **THEN** 學生端 MUST 回到名單選擇畫面
- **AND** 學生 MUST 需重新選擇可用名單項目才能再次加入
