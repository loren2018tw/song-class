## MODIFIED Requirements

### Requirement: 學生端必須先提交暱稱才能加入會話

系統 MUST 在學生端瀏覽器 Vue 頁面首次載入時顯示班級名單選擇流程，並在學生選定有效名單項目前禁止進入後續會話。

#### Scenario: 首次連線顯示班級學生名單

- **WHEN** 學生使用瀏覽器開啟教師端提供的連線網址
- **THEN** 頁面 MUST 顯示目前班級學生名單按鈕
- **AND** 系統 MUST 在未選定名單項目前維持未加入狀態

#### Scenario: 選定有效名單項目時完成加入

- **WHEN** 學生選擇可用的班級學生按鈕並送出加入
- **THEN** 系統 MUST 建立該學生名單項目的會話身分並回傳加入成功
- **AND** 系統回傳或後續顯示的學生識別文字 MUST 使用「座號暱稱（無分隔）」格式（例如 `01王小明`）

### Requirement: 系統必須驗證暱稱輸入有效性

系統 MUST 驗證學生加入請求必須對應目前班級中一個存在且可用的名單項目，並對無效或已占用項目提供可理解的錯誤提示。

#### Scenario: 選到不存在或無效名單項目

- **WHEN** 學生送出一個不屬於目前班級的名單項目
- **THEN** 系統 MUST 拒絕加入請求
- **AND** 頁面 MUST 顯示需選擇有效班級學生的提示

#### Scenario: 名單項目已被占用

- **WHEN** 學生送出已在線占用的名單項目
- **THEN** 系統 MUST 拒絕加入請求
- **AND** 頁面 MUST 顯示該學生已連入不可重複加入的提示

#### Scenario: 驗證失敗後可重新提交

- **WHEN** 學生收到名單驗證失敗訊息
- **THEN** 頁面 MUST 保持在名單選擇流程
- **AND** 學生 MUST 可重新選擇並再次送出
