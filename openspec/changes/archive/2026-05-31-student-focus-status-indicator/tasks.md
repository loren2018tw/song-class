## 1. 事件契約與型別擴充

- [x] 1.1 在課堂即時事件型別中新增學生分頁焦點狀態事件（含學生識別與狀態值）。
- [x] 1.2 更新教師端與學生端共用的 session 型別，加入學生專心狀態欄位與預設值。
- [x] 1.3 確認舊版端點未送出焦點事件時，事件解析與連線流程仍可正常運作。

## 2. 學生端焦點狀態上行

- [x] 2.1 在學生端註冊 `visibilitychange` 監聽，將可見性映射為「專心中 / 已離開分頁」。
- [x] 2.2 在學生加入成功、切離分頁、切回分頁與重連後送出最新焦點狀態事件。
- [x] 2.3 實作最小去抖/防抖策略，避免短時間切換造成過量事件。

## 3. 教師端狀態儲存與清單顯示

- [x] 3.1 在教師端收到焦點狀態事件時，以同一學生最後一筆事件覆寫目前狀態。
- [x] 3.2 調整連線清單學生暱稱標籤底色映射，明確區分專心中與離開分頁兩種狀態。
- [x] 3.3 定義離線優先顯示規則，確保離線狀態不被專心狀態覆蓋。

## 4. 驗證與回歸測試

- [x] 4.1 驗證學生切離/切回分頁時，教師端顏色標籤可即時更新。
- [x] 4.2 驗證學生斷線與重連流程下，教師端狀態顯示符合預期且不殘留舊狀態。
- [x] 4.3 驗證未支援焦點事件的舊端點仍可加入課堂並維持既有功能。
