## Context

目前老師介面有教學、白板、清單等功能，但缺乏一種「待機/提醒」模式，用於在非正式教學期間（如轉場、休息、移動中）同步資訊給學生。本設計旨在新增一個「提醒看板」模組。

## Goals / Non-Goals

**Goals:**

- 提供直觀的全螢幕提醒介面，包含時間、日期與大型標題。
- 支援快速切換預設看板類別（移動、作息）。
- 提供自訂看板功能，並將資料持久化於資料庫。
- 確保所有內建看板內容易於在代碼中調整。

**Non-Goals:**

- 不涉及複雜的排程自動切換功能（目前僅由老師手動切換）。
- 暫不支撐多媒體（影片/音訊）背景，以純色與圖示為主。

## Decisions

### 1. 視圖架構 (Vue Component)

- **`ReminderBoardView.vue`**：主入口視圖。
- **時鐘組件**：使用 `setInterval` 每秒更新一次時間與日期，格式包含星期（如：6月2日 星期二）。
- **看板展示區**：使用 `v-bind` 動態渲染當前選中的標題、副標題與圖示。
- **語音朗讀與導航菜單**：
  - 左側新增喇叭圖示按鈕（切換語音開/關）。
  - 當語音開啟且看板內容變更時，觸發 Web Speech API (SpeechSynthesis) 朗讀標題與副標題。
  - 中間為類別切換邏輯。
  - 右側為「設定」圖示進入編輯頁。

### 2. 資料結構與預設值

- **預設看板 (In-code Constants)**：將「移動」與「作息」的內容定義在 `src/constants/reminderBoards.ts` 中，這些看板不儲存於資料庫，直接由程式讀取。
- **自訂看板 (Database)**：
  - 資料表 `reminder_boards`: `id`, `category` (固定為 'custom'), `title`, `subtitle`, `icon`。
  - 使用 SQL Migration (`next_release.sql`) 初始化資料表。

### 3. UI/UX 套件

- 統一使用 **Vuetify 3** 元件。
- 圖示使用 **Material Design Icons (MDI)**。
- 導航欄使用 `v-btn-toggle` 或 `v-tabs` 來實現分類切換。

## Risks / Trade-offs

- **[Risk] 時間同步性**：前端時鐘若由老師端本地時間驅動，可能與學生端或實際時間有微小誤差。
  - **Mitigation**：此功能定位為「提醒」而非精確授課計時，微小誤差（秒級）可接受。
- **[Risk] 使用者體驗**：全螢幕模式下可能遮蔽重要通知。
  - **Mitigation**：在 UI 邊角提供明顯的「關閉/返回」按鈕（如右上角的 X）。
- **[Trade-off] 預設值維護**：將部分看板寫死在代碼中。
  - **Reason**：為了保持簡單並符合使用者「手動隨時修改」的需求，初期使用代碼變數比複雜的後端 UI 更直觀。
