## Why

教師在課堂互動中需要快速記錄個別與小組表現，但目前系統缺少可即時加減分與重設分數的教學管理工具，導致流程仰賴外部工具且難以與既有名單、座號與暱稱整合。現在導入學生積點模組可直接在教師端完成評量與分組管理，並與現有課堂即時互動體驗一致。

## What Changes

- 教師端新增「學生積點模組」入口，並以標籤分頁提供「個人積點」與「小組模式」。
- 個人積點分頁提供每位學生積點卡片，支援單人 +1/-1，並提供「全班+1」「全班-1」「積點重設（含確認對話框）」。
- 小組模式分頁提供上方「分幾組」輸入框、左側學生清單、右側群組區與「未分組」區塊；支援拖拉即時分組，並可對整組所有成員執行 +1/-1。
- 分組與積點資料統一儲存在學生資料表，新增「第幾組」與「積分」欄位，分組畫面由學生資料分群即時推導，不需另存分組設定檔。
- 當「分幾組」調小時，被移除組別中的學生必須自動移入「未分組」區塊。
- 小組拖拉調整後需即時更新學生組別，不提供額外儲存按鈕。
- 涉及資料結構調整時，以命令列方式修改並記錄於 `src-tauri/sql/migrations/next_release.sql`，供後續正式 release migration 使用。

## Capabilities

### New Capabilities

- `student-point-individual-scoring`: 教師可對個別學生與全班進行積點加減、重設與狀態呈現。
- `student-point-group-mode`: 教師可拖拉分組、管理各組成員，並對整組套用積點加減。

### Modified Capabilities

- （無）

## Impact

- 前端教師頁面：`src/views/WebTeacherView.vue` 及可能抽離的新 Vue 元件（Vuetify）。
- 前端型別與狀態：可能新增積點與分組相關型別於 `src/types/`。
- Rust 後端與資料持久層：新增或調整積點資料結構與 API/訊息流。
- 學生資料表：新增「第幾組」「積分」欄位，作為小組與積點唯一資料來源。
- 資料庫 migration：若有 schema 變更，更新 `src-tauri/sql/migrations/next_release.sql`，不在本變更中直接實作程式碼。
