## ADDED Requirements

### Requirement: 教師端必須提供小組模式分頁與拖拉分組介面

系統必須在學生積點模組中提供「小組模式」分頁，包含上方分組數輸入框、左側學生清單、右側群組區塊與未分組區塊，教師可將學生拖拉到任一群組或未分組。

#### Scenario: 將學生拖拉到群組

- **WHEN** 教師把左側學生項目拖拉並放置到某一群組區塊
- **THEN** 系統必須更新該學生的群組歸屬並在介面即時呈現

#### Scenario: 將學生拖拉到未分組

- **WHEN** 教師把學生拖拉並放置到未分組區塊
- **THEN** 系統必須將該學生標記為未分組並在介面即時呈現

### Requirement: 群組區塊必須支援整組積點加減

系統必須在每個群組區塊提供「+1」與「-1」操作，並對該群組內每位成員分別執行 +1 或 -1。

#### Scenario: 對群組全員加 1 點

- **WHEN** 教師按下某群組區塊的「+1」按鈕
- **THEN** 系統必須將該群組所有成員積點各增加 1 點

#### Scenario: 對群組全員減 1 點

- **WHEN** 教師按下某群組區塊的「-1」按鈕
- **THEN** 系統必須將該群組所有成員積點各減少 1 點

### Requirement: 小組模式必須支援即時調整分組數量

系統必須在小組模式上方提供「分幾組」輸入框，教師可隨時調整組數；當組數減少時，被移除組別的學生必須自動移至未分組區塊。

#### Scenario: 調高分組數量

- **WHEN** 教師將分組數量由 N 調高為 M（M > N）
- **THEN** 系統必須即時新增對應群組區塊

#### Scenario: 調低分組數量

- **WHEN** 教師將分組數量由 N 調低為 M（M < N）
- **THEN** 系統必須將原本組號大於 M 的學生移至未分組區塊

### Requirement: 分組與積點必須使用學生資料表欄位

系統必須在學生資料表使用「第幾組」與「積分」欄位作為小組與分數資料來源，並以學生資料分群推導目前組別；積分允許負值，未分組統一以 0 表示。

#### Scenario: 載入小組模式資料

- **WHEN** 教師開啟小組模式分頁
- **THEN** 系統必須從學生資料表讀取「第幾組」與「積分」欄位並完成分群顯示

### Requirement: 小組編輯必須即時儲存

系統必須在拖拉學生變更組別時立即更新學生組別資料，不提供額外儲存按鈕。

#### Scenario: 拖拉後即時更新

- **WHEN** 教師將學生拖拉到不同組塊
- **THEN** 系統必須立即更新該學生組別欄位且不需要點擊儲存

### Requirement: 資料結構異動必須以 migration 草稿檔記錄

若積點模組需要調整資料結構，系統開發流程必須透過命令列完成異動，並將內容寫入 `next_release.sql` 供後續 release 升級使用。

#### Scenario: 需要新增或調整資料表欄位

- **WHEN** 開發者判定積點模組需要資料結構異動
- **THEN** 開發者必須以命令列修改並將異動寫入 `src-tauri/sql/migrations/next_release.sql`
