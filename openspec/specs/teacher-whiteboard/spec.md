## Purpose

定義教師端小白板元件的功能需求，包括切換入口、畫布縮放、輸入支援與資料輸出，確保元件可在不同容器尺寸下正確呈現，且支援遠端同步。

## Requirements

### Requirement: 教師端必須提供小白板切換入口與主畫面配置

教師端頁面 MUST 在學生清單上方提供「小白版」功能按鈕，並在切換後顯示小白板主畫面。

#### Scenario: 教師切換至小白板模式

- **WHEN** 教師在教師端頁面點擊「小白版」按鈕
- **THEN** 系統 MUST 切換到小白板主畫面
- **AND** 系統 MUST 將學生清單保留在左側區域
- **AND** 系統 MUST 將小白板顯示在右側主要內容區

#### Scenario: 教師回到學生清單區域時仍可看見白板內容

- **WHEN** 教師在同一頁面瀏覽學生清單與功能按鈕列
- **THEN** 系統 MUST 保持上方為功能按鈕列
- **AND** 系統 MUST 保持下方左側為學生清單
- **AND** 系統 MUST 保持下方右側為目前功能主畫面區

### Requirement: 小白板元件必須使用固定內部畫布尺寸並可縮放顯示

小白板元件 MUST 以 1280×720 px 作為固定內部畫布座標系，並依外層容器大小縮放呈現。

#### Scenario: 元件在不同容器大小下顯示

- **WHEN** 小白板元件被放入不同寬高的顯示區域
- **THEN** 系統 MUST 以相同的 1280×720 內部座標繪製內容
- **AND** 系統 MUST 將畫布依外層區域大小縮放顯示

#### Scenario: 同一筆畫在縮放後仍維持相同相對位置

- **WHEN** 內容已在固定畫布座標系中完成塗繪
- **THEN** 系統 MUST 在任何縮放比例下維持筆畫的相對位置與比例

### Requirement: 小白板元件必須支援滑鼠與觸控塗繪

小白板元件 MUST 支援滑鼠與觸控輸入進行連續塗繪，並提供畫筆顏色、畫筆粗細、橡皮擦與清除畫布功能。

#### Scenario: 使用滑鼠繪圖

- **WHEN** 使用者按住滑鼠並在畫布上拖曳
- **THEN** 系統 MUST 以目前畫筆顏色與粗細繪製連續線段

#### Scenario: 使用觸控繪圖

- **WHEN** 使用者以觸控筆或手指在畫布上拖曳
- **THEN** 系統 MUST 以與滑鼠相同的方式繪製連續線段

#### Scenario: 切換橡皮擦

- **WHEN** 使用者點選橡皮擦按鈕
- **THEN** 系統 MUST 將畫筆顏色改為白色
- **AND** 系統 MUST 將畫筆粗細自動調整為目前粗細再加 10 點

#### Scenario: 清除畫布

- **WHEN** 使用者點選清除畫布功能
- **THEN** 系統 MUST 清空目前所有塗繪內容

### Requirement: 小白板元件必須輸出可重現的 JSON 塗繪資料

小白板元件 MUST 將塗繪內容整理為可序列化 JSON，並透過事件傳送給父元件，以便後續傳送到遠端元件或重新還原內容。

#### Scenario: 塗繪內容更新時向父元件回傳資料

- **WHEN** 使用者完成一段塗繪、切換工具或清除畫布
- **THEN** 小白板元件 MUST 透過事件回傳最新 JSON 塗繪資料給父元件

#### Scenario: 遠端元件還原塗繪內容

- **WHEN** 父元件將相同 JSON 塗繪資料提供給另一個小白板元件實例
- **THEN** 該元件 MUST 還原出相同的塗繪內容

#### Scenario: JSON 內容可供後續傳送

- **WHEN** 父元件取得小白板輸出的 JSON
- **THEN** 該資料 MUST 可直接作為後續同步或儲存的輸入
